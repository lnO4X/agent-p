# AI+UGC 剧本杀平台 — 防御性分析

> 基于 `ai-product-defensibility-framework.md` 八维评估矩阵
> 评估日期: 2026-03-17

---

## 产品概念

**核心定位**: AI驱动的UGC剧本杀平台 — 任何人可以创建剧本，AI动态扮演NPC，玩家在线/线下组局。

**技术复用**: 复用GameTan的AI伙伴系统（四层prompt架构、记忆系统、知识图谱），扩展为多角色协同。

---

## 八维防御性评分

| 维度 | 得分 | 理由 |
|------|------|------|
| **数据独占性** | 2 | UGC剧本库 + 玩家行为数据 + 角色表演数据 → 独有数据集。OpenAI无法获取这些领域特定的创作和交互数据。每个剧本是创作者的IP，平台独占发布权。 |
| **工作流深度** | 2 | 从创作到发布到组局到游玩到复盘，全链路闭环。DM工具 + 组局系统 + AI NPC引擎 + 语音房 → 多系统深度集成。替换任一环节都很痛苦。 |
| **网络效应** | 2 | 三重网络效应：(1) 创作者越多，剧本越丰富；(2) 玩家越多，组局越快；(3) AI角色表演数据越多，NPC越智能。典型双边市场 + 数据网络效应。 |
| **切换成本** | 2 | 创作者已发布的剧本库、粉丝、评分不可迁移。玩家的游玩记录、信誉评分、好友关系也不可迁移。DM积累的导演经验和模板形成隐性切换成本。 |
| **垂直深度** | 2 | 极深垂直：剧本杀是有独特规则的社交推理游戏，需要专业领域知识（剧本结构、角色平衡、线索设计、反转机制）。通用AI平台做不到这种深度。 |
| **大厂风险** | 1 | OpenAI/Google不太可能专门做剧本杀平台（太垂直），但可能推出通用RPG工具。中国市场有百变大侦探、我是谜等竞品，但它们尚无成熟的AI NPC系统。给1分而非2分是因为中国厂商可能快速跟进。 |
| **利润可行性** | 2 | AI成本可控：NPC对话可用轻量模型（Gemini Flash级别），单局约10-30轮对话。可缓存常见剧情分支。主要成本是创作者分成。平台抽成模式（Steam式30%或更低），UGC内容无采购成本。 |
| **数据飞轮** | 1 | 飞轮存在但早期需要启动量：玩家反馈 → 优化NPC表演 → 更好体验 → 更多玩家。创作者评分系统帮助筛选高质量剧本。但需要~1000+活跃用户才能让飞轮真正转起来。 |

**总分: 14/16** — 强防御性，建议全力投入。

---

## 关键壁垒分析

### 为什么强防御？

1. **UGC内容库不可复制** — 创作者花费数小时设计的剧本+角色+线索形成独有IP库。这不是API调用可以替代的。

2. **三边市场** — 创作者(supply) × 玩家(demand) × DM/主持人(facilitator)。三边网络效应比双边更难攻破。

3. **领域知识壁垒** — 剧本杀有复杂规则（密封本/盒装本区别、推理本/情感本/恐怖本分类、角色平衡设计、反转时机）。这种领域知识必须深度编码到产品中。

4. **社交图谱锁定** — 玩家的固定局友、信任DM、游玩历史形成强社交锁定。

### 与GameTan的协同

| GameTan资产 | 在剧本杀中的复用 |
|-------------|----------------|
| 四层prompt架构 | NPC角色扮演的核心引擎 |
| 记忆系统 | NPC记住本局线索和玩家行为 |
| 知识图谱 | 跨场景的角色关系追踪 |
| 天赋系统 | 玩家推理能力评分 → 匹配适合难度的剧本 |
| 对话摘要 | 长剧本的上下文压缩 |

## 风险因素

| 风险 | 严重度 | 缓解措施 |
|------|--------|---------|
| 冷启动（需要足够剧本+玩家才能组局） | 高 | 初期自建10-20高质量种子剧本，邀请头部创作者入驻 |
| 语音/实时通信复杂度 | 中 | 初期文字聊天为主，语音房Phase 2接入 |
| 中国市场监管（内容审核） | 低 | 产品不在中国运营，海外华人+国际市场 |
| AI NPC不够"智能" | 中 | 专注规则遵守而非创意对话；用专门的剧本杀prompt template |

---

## 建议的MVP范围

**Phase 1 (2-3周)**:
1. 剧本创作工具（Markdown编辑器 + 角色/线索模板）
2. AI NPC引擎（复用GameTan四层prompt，增加角色约束层）
3. 文字组局系统（房间 + 阶段推进 + 投票）
4. 基础UGC发布 + 评分

**Phase 2 (月2-3)**:
- 语音集成
- 创作者分成
- 推荐算法

## 对比评分

| 产品 | 防御性得分 | 阶段 |
|------|-----------|------|
| GameTan (当前) | 6/16 | 需强化 |
| AI+UGC剧本杀 | 14/16 | 强防御 |
| 纯AI陪聊 (Replika模式) | 4/16 | 极弱 |
| AI游戏推荐 (通用) | 3/16 | 无壁垒 |

**结论**: AI+UGC剧本杀作为独立产品，防御性远优于GameTan当前方向。建议作为独立项目启动，复用AI伙伴技术栈。
