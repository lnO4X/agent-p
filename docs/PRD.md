# Agent-P —— 游戏搭子众包平台

> AI Agent 驱动的游戏陪玩匹配平台

## 1. 产品定位

**Agent-P** 是一个游戏陪玩众包平台，连接想找人一起玩的「玩家」和提供陪玩服务的「搭子」。平台以 AI Agent 对话为核心交互方式，用户通过自然语言发布需求、接单、管理订单，辅以表单确认和滑动匹配等 UI 组件。

### 产品名称说明

| 元素 | 含义 |
|------|------|
| **Agent** | AI Agent 驱动，也代表"经纪人"——帮你找到最佳搭子 |
| **P** | Play / Partner / Platform |
| 英文项目名 | `agent-p` |

## 2. 用户角色

### 2.1 平台管理员（Admin）

平台的超级管理者，拥有最高权限。

**核心能力：**

- 管理所有用户、公会、订单、资金
- 维护标准项目库（如王者荣耀、和平精英等），为每个标准项目设置平台最低价格
- 审核公会注册、搭子资质
- 配置平台全局参数（结算周期、佣金比例、邀请奖励比例等）
- 处理争议与投诉
- 未来方向：由 AI Agent 辅助或替代部分管理工作

### 2.2 公会（Guild）

组织和管理搭子的中间层。

**核心能力：**

- 注册后经平台审核生效
- 从标准项目库中选择开启哪些项目
- 创建自定义项目（冷门游戏、娱乐项目等），上传项目图片、名称、设置费用
- 可按段位/级别为项目设置高于平台最低价的价格（不设置则沿用平台价格）
- 登记和管理旗下搭子
- 当前阶段：平台仅有 1 个公会（未来扩展多公会）

### 2.3 搭子（Buddy）— 服务提供方

提供陪玩服务的个人用户。

**注册流程：**

- 个人注册 → 完善必要信息（昵称、头像、简介、联系方式等）
- 选择角色为「搭子」
- 选择擅长的项目、技能、段位
- 自由选择加入某个公会

**核心能力：**

- 浏览匹配的订单（Tinder 式滑动）：看到符合自己技能/段位的订单
- 左滑跳过 → 出现下一个订单
- 右滑接单 → 开启与玩家的聊天
- 点击「准备好了」按钮通知玩家
- 完成服务后等待玩家确认

### 2.4 玩家（Player）— 服务消费方

花钱找搭子一起玩的用户。（对应口语中的"老板"）

**注册流程：**

- 个人注册 → 完善必要信息
- 选择角色为「玩家」
- 充值后方可消费

**核心能力：**

- 通过 AI Agent 对话发布需求（如："我想打王者了"）
- Agent 引导确认：几局、段位要求、出价
- 也可以 Tinder 式浏览已接单的搭子，左右滑动选择
- 确认搭子「准备好了」后开始游戏
- 游戏结束后确认完成，触发结算记账

## 3. 核心业务流程

### 3.1 发单与匹配流程

```
玩家 → AI Agent 对话 → "我想打王者了"
                         ↓
              Agent 引导确认（表单 UI）：
              - 项目：王者荣耀
              - 局数：2局
              - 段位要求：钻石以上
              - 出价：每局 15 元（≥ 平台最低价 10 元）
                         ↓
              生成需求订单，进入匹配池
                         ↓
        ┌────────────────┴────────────────┐
        ↓                                  ↓
  搭子端：Tinder式浏览          玩家端：Tinder式浏览
  看到匹配的订单                已接单的搭子列表
  左滑跳过 / 右滑接单           左滑跳过 / 右滑选择
        ↓                                  ↓
        └────────────────┬────────────────┘
                         ↓
                  双向匹配成功
                         ↓
                  开启 1v1 聊天
                         ↓
              搭子点击「准备好了」
                         ↓
              玩家点击「确认开始」
                         ↓
                    进行游戏
                         ↓
              玩家确认「游戏完成」
                         ↓
                  记账，进入结算队列
```

### 3.2 结算流程

- 玩家确认完成后，系统**记账**（不实时转账）
- 按固定周期清算（默认每日 1 次）
- 平台抽取佣金后，剩余金额记入搭子可提现余额
- 佣金比例由管理员配置

### 3.3 资金入口（待定/重点难题）

**背景：** 平台注册在中国境外，需支持中国境内用户入金。

**方案探讨：**

| 方案 | 说明 | 可行性 |
|------|------|--------|
| C2C 交易 | 类似币安 P2P，用户间直接转账，平台撮合确认 | ⭐ 最可行，合规风险需评估 |
| 第三方跨境支付 | 如 Stripe（不支持中国大陆）、Airwallex、PingPong | 需具体调研资质要求 |
| USDT/稳定币充值 | 加密货币入金，平台内以 RMB 计价 | 技术可行，合规灰色地带 |
| 代充模式 | 线下/微信转账给代理，代理在平台加余额 | 早期可用，不可规模化 |

> **当前决策：** 优先实现 C2C 撮合模式，保留接口扩展其他支付方式。初期可用代充模式过渡。

## 4. 项目与定价体系

### 4.1 标准项目（平台维护）

由管理员创建和维护的主流游戏项目。

- 项目信息：名称、图标、描述、分类
- 平台最低价：每个标准项目设定最低单价（单位：RMB/局 或 RMB/小时）
- 段位体系：可按段位分级

### 4.2 公会自定义项目

公会可创建平台未收录的项目。

- 上传项目图片、名称
- 设置费用（不低于平台最低价，若未设置沿用平台价）
- 可按段位/级别细分价格
- 适用于冷门游戏或娱乐项目

### 4.3 定价规则

```
最终价格 ≥ max(平台最低价, 公会最低价)
玩家出价 ≥ 最终价格 → 订单有效
```

## 5. AI Agent 交互设计

### 5.1 设计理念

平台核心交互以大模型对话为主，而非传统表单页面。Agent 在对话中智能唤起 UI 组件。

### 5.2 Agent 可唤起的 UI 组件

| 组件 | 场景 |
|------|------|
| 表单确认卡片 | 发单确认（项目、局数、出价） |
| 滑动卡片（Tinder） | 浏览订单/搭子 |
| 聊天窗口 | 匹配成功后的 1v1 沟通 |
| 状态按钮 | 「准备好了」「确认开始」「游戏完成」 |
| 支付/充值弹窗 | 余额不足时引导充值 |
| 个人资料编辑 | 修改技能、段位、简介 |

### 5.3 对话示例

**玩家发单：**

```
玩家：我想打王者了
Agent：好嘞！想打几局？对搭子段位有要求吗？
玩家：来两把吧，钻石以上就行
Agent：当前王者荣耀最低 10 元/局，你想出多少？
玩家：就 10 块一局
Agent：[弹出确认卡片]
       📋 订单确认
       项目：王者荣耀
       局数：2 局
       段位要求：钻石+
       单价：10 元/局
       总计：20 元
       [确认发布] [修改]
```

**搭子接单：**

```
Agent：有新订单匹配你的段位哦！
       [滑动卡片]
       🎮 王者荣耀 × 2局
       💰 10元/局
       📊 要求：钻石+
       ← 跳过    接单 →

搭子：[右滑接单]
Agent：已接单！正在等玩家确认...
       [玩家确认后]
Agent：匹配成功！已开启聊天 💬
```

## 6. 邀请机制

### 6.1 基本规则

- 每个用户拥有唯一邀请码/链接
- 被邀请人注册时绑定邀请关系（永久绑定）

### 6.2 赏金规则

- 被邀请人选择「玩家」角色并充值时，邀请人获得充值金额的 X% 作为赏金
- **永久有效**：被邀请的玩家每次充值，邀请人都获得分成
- 赏金比例由管理员配置（如 5%）
- 赏金从平台佣金中扣除，不额外增加玩家成本

### 6.3 示例

```
A 邀请 B 注册 → B 选择玩家角色 → B 充值 100 元
→ A 获得 5 元赏金（假设比例 5%）
→ 之后 B 每次充值，A 都获得 5% 赏金
```

## 7. 社交与扩展

### 7.1 好友系统（预留）

- 完成订单后，双方**不能**直接添加好友
- 预留好友功能的技术接口
- 未来作为付费功能开放（如：付费解锁添加好友）

### 7.2 评价系统

- 游戏结束后双方可互评
- 评价维度：技术水平、态度、准时度
- 评分影响搭子排名和曝光

## 9. 阶段规划

### Phase 1 — MVP

- 单公会模式
- 标准项目 + 自定义项目
- AI Agent 对话发单
- Tinder 式匹配
- 基础聊天
- 记账 + 手动/代充入金
- 邀请机制

### Phase 2 — 支付与结算

- C2C 入金系统
- 自动化日结算
- 提现功能
- 争议处理流程

### Phase 3 — 多公会与运营

- 多公会注册与管理
- 评价体系
- 搭子等级/排行
- 运营工具（活动、优惠券）

### Phase 4 — Agent 驱动

- AI Agent 辅助平台管理（自动审核、异常检测）
- Agent 驱动的智能匹配
- 付费好友功能
- 增值服务

## 10. 关键约束与风险

| 风险 | 说明 | 应对 |
|------|------|------|
| 跨境支付 | 境外平台接受境内资金 | 优先 C2C 模式，合规评估 |
| 资金安全 | 无担保交易 | 评价体系 + 争议处理 + 信用分 |
| 合规 | 众包平台法律风险 | 注册地法律咨询 |
| 冷启动 | 早期供需不平衡 | 单公会运营，集中资源 |
| Agent 体验 | 大模型响应质量 | 预设场景模板 + 兜底 UI |
